# SevaSetu – Request Autofill AI Microservice

> **AI-powered form autofill for NGO help requests and volunteer task posts.**
> Paste any free-text description and the AI pre-fills category, urgency, location, required skills, volunteer count, deadline, and more.

---

> ⚠️ **AI Autofill is assistive only. Always review all autofilled fields before submitting a request.**

---

## How It Works

The service uses a **hybrid NLP approach**:

| Layer | What it does |
|---|---|
| **TF-IDF (word + char n-grams)** | Converts request text into feature vectors |
| **LightGBM classifiers** | Predicts categorical fields: `category`, `subcategory`, `urgency_level`, `effort_level`, `shift_type`, etc. |
| **Logistic regression** | Predicts boolean flags: `is_medical_emergency`, `transport_required`, etc. |
| **Regex / rule extractors** | Extracts entities: city, volunteer count, people affected, deadline hours, skills, items |
| **OR-fusion** | Boolean fields fuse ML prediction + rule signals for higher recall |

## Training the Model

**Step 1 – Generate synthetic dataset** (optional, train.py does this automatically if data is missing):
```bash
python generate_dataset.py
# Saves 3200 rows to data/dataset.csv
```

**Step 2 – Train all models:**
```bash
python train.py
# Trains TF-IDF + LightGBM for each output field
# Saves models/autofill_model.pkl
```

Expected output:
```
Loaded 3200 rows
Fitting TF-IDF …
Training categorical classifiers …
  [category]           acc=0.91  f1=0.89  classes=[...]
  [subcategory]        acc=0.87  f1=0.85  ...
  [urgency_level]      acc=0.88  f1=0.86  ...
Training boolean classifiers …
  [is_medical_emergency]  acc=0.93
  [is_disaster_related]   acc=0.94
  ...
✅  Model saved → models/autofill_model.pkl
```

```json
  "message": "Model loaded and ready."

```


### `POST /autofill/batch`

Autofill up to 20 descriptions at once.

**Request:**
```json
{
  "descriptions": [
    "Need 3 volunteers with bikes to deliver medicine in Pune tonight",
    "Flood relief for 50 people in Nashik, need food and blankets urgently"
  ]
}
```

## Confidence Scores

Every response includes a `confidence` dictionary with per-field confidence (0.0–1.0):

- **> 0.85** → High confidence, likely accurate
- **0.65–0.85** → Medium confidence, review recommended  
- **< 0.65** → Low confidence, manual entry recommended
